# 长期持有研究续做记忆

更新时间：2026-04-17

## 1. 当前研究边界

- 本阶段只做“数据抓齐、抓对”的准备，不做量化打分，不做公司分类结论。
- 当前不研究市场验证历史，也不研究机构“买得对不对”。
- 当前研究问题聚焦为：
  - 哪些公司真的被长期持有了？
  - 哪些公司具有持续经营质量，值得长期持有？

## 2. 已确认的方法方向

- 后续研究主轴不是“收益验证”，而是：
  - `持有持续性`
  - `经营持续性`
- 持仓强度优先使用“占各自组合的持仓比例/权重”，不要只看绝对持仓金额。
- 后续如果要做四象限或其他分类，必须建立在可信数据层之上，不能直接用当前快照硬分。
- 研究目标不是做一个筛选器，而是建立一套“长期持有审美”的量化研究框架：
  - 哪些公司值得长期持有
  - 哪些公司也确实被长期持有了
  - 哪些公司曾经符合这种审美，但后来掉队了

## 3. 研究三层结构

```text
第1层：可信数据层
- 实体级持仓历史
- 年度经营质量时序
- 字段验真规则

第2层：量化研究层
- 持有持续性分数
- 经营持续性分数
- 观察区规则

第3层：案例研究层
- 四类公司样本池
- 每类研究问题
- 长期持有审美归纳
```

## 5. 未来打分框架

### 5.1 持有持续性分数

用于衡量“这些公司是否真的被长期持有”。

建议优先纳入的字段：
- `覆盖机构数`
- `出现季度数`
- `覆盖年份数`
- `平均持仓权重`
- `峰值持仓权重`
- `当前是否仍持有`
- `当前持有机构数`

方法原则：
- 时间维度为主，权重维度为辅
- `position_weight` 比绝对金额更重要
- 当前仍持有只给小权重加分，避免把短期状态过度混入历史认同

候选权重框架：
- `出现季度数` 30%
- `覆盖年份数` 20%
- `覆盖机构数` 20%
- `平均持仓权重` 15%
- `峰值持仓权重` 10%
- `当前是否仍持有` 5%

### 5.2 经营持续性分数

用于衡量“公司是否持续值得长期持有”。

建议先拆成两个子分数：
- `经营质量水平分`
- `经营质量稳定分`

之后合成为：
- `经营持续性 = 60% 质量水平 + 40% 稳定性`

候选输入字段：
- `Revenue`
- `ROE`
- `ROA`
- `Gross Margin`
- `Net Margin`
- `Debt/Equity`
- `FCF`
- `FCF Margin`
- `Share Count`
- `Dividend`

稳定性优先关注：
- 5年均值
- 5年波动
- 趋势方向
- 是否出现明显塌陷

## 6. 未来分类框架

主结构：
- 横轴：`持有持续性`
- 纵轴：`经营持续性`

四类定义：
- `A类：长期赢家`
  - 持有持续性高 + 经营持续性高
- `B类：审美失效 / 退化案例`
  - 持有持续性高 + 经营持续性低
- `C类：经营优秀但未被长期拿住`
  - 持有持续性低 + 经营持续性高
- `D类：弱质 / 陷阱样本`
  - 持有持续性低 + 经营持续性低

额外保留：
- `观察区`
  - 中间分位样本，不强行归入四类

注意：
- 下次做分类时，不要用当前快照直接硬分
- 必须先拿到经营持续性所需的年度历史数据

## 7. 分类后怎么继续研究

分类不是结论，只是选样本。

后续每类的研究重点：

- `长期赢家`
  - 研究哪些特征使其长期值得持有，而且也真的被长期持有
- `审美失效 / 退化案例`
  - 研究什么信号意味着曾经的好公司开始失去长期持有价值
- `经营优秀但未被长期拿住`
  - 研究为什么经营上不错，但没有进入这些机构的长期审美
- `弱质 / 陷阱样本`
  - 研究哪些低估或热门叙事不支持长期持有

后续研究输出建议分两层：
- `全池分类总表`
  - 持有持续性分数
  - 经营持续性分数
  - 分类结果
  - 是否观察区
  - 当前是否仍持有
  - 研究优先级
- `案例研究表`
  - 为什么被长期持有
  - 经营持续性表现如何
  - 属于哪一类
  - 值得长期持有的关键特征
  - 失去长期持有价值的关键信号

# 长期持有研究数据字典与验真框架

更新时间：2026-04-17

## 1. 本文档用途

本文档只解决下一阶段“数据抓齐、抓对”的问题，不负责量化打分、公司分类或投资结论。

目标是为“哪些公司值得长期持有，并且真的被长期持有了”这一研究问题准备可信数据层。

## 2. 数据层范围

下一阶段数据只分两块：

1. `实体级持仓历史`
2. `年度经营质量历史`

不纳入当前阶段：
- 市场验证历史
- 收益归因
- 回测
- 四象限分类
- 公司优劣结论

## 3. 实体级持仓历史数据字典

### 3.1 核心主键

建议主键：
- `manager_entity`
- `ticker`
- `report_date`

辅助识别字段：
- `issuer_name_raw`
- `issuer_norm`
- `cusip`
- `title_class`
- `source`

### 3.2 核心字段

| 字段 | 含义 | 单位/格式 | 是否必需 | 备注 |
|---|---|---|---|---|
| `manager_group` | 机构家族桶，如巴芒系 | 文本 | 是 | 仅作展示，不作最终当前持有判断 |
| `manager_entity` | 实体管理人 | 文本 | 是 | 必须区分 Berkshire / Daily Journal / Himalaya / HHLR |
| `ticker` | 机构实际持有交易代码 | 文本 | 是 | 不强行切换到本土主上市代码 |
| `report_date` | 13F 报告对应持仓日期 | 日期 | 是 | 例如季度末 |
| `filing_date` | 申报日期 | 日期 | 是 | 用于时点核对 |
| `value_usd_thousand` | 13F 原始市值 | 千美元 | 是 | 保留原始字段，防止单位漂移 |
| `value_usd` | 标准化市值 | 美元 | 是 | `value_usd_thousand * 1000` |
| `shares` | 持股数 | 股/份 | 是 | 保留原始披露口径 |
| `position_weight` | 该标的占该实体组合的权重 | 比例 | 是 | 当前阶段优先研究字段 |
| `is_first_seen` | 是否首次进入该实体组合 | 布尔 | 否 | 派生字段 |
| `is_current` | 是否仍在最新组合中 | 布尔 | 是 | 基于该实体最新 13F 判断 |
| `quarters_held_by_entity` | 该实体持有季度数 | 整数 | 否 | 派生字段 |
| `years_held_by_entity` | 该实体持有年份数 | 整数 | 否 | 派生字段 |

### 3.3 强度字段优先级

研究“长期持有审美”时，优先级如下：

1. `position_weight`
2. `quarters_held_by_entity`
3. `years_held_by_entity`
4. `is_current`
5. `value_usd`

说明：
- `value_usd` 只能表示规模，不能单独表示偏好强度
- `position_weight` 更能反映该实体的真实重视程度

## 4. 年度经营质量历史数据字典

### 4.1 最小必需字段集

| 字段 | 含义 | 单位/格式 | 是否必需 | 研究用途 |
|---|---|---|---|---|
| `fiscal_year` | 财年 | 年份 | 是 | 年度主键 |
| `revenue` | 收入 | 原币种金额 | 是 | 经营规模与增长 |
| `gross_margin` | 毛利率 | 比例 | 是 | 商业模式质量 |
| `operating_margin` | 营业利润率 | 比例 | 建议 | 盈利结构 |
| `net_margin` | 净利率 | 比例 | 是 | 盈利质量 |
| `roe` | 净资产收益率 | 比例 | 是 | 资本效率 |
| `roa` | 总资产回报率 | 比例 | 是 | 资产效率 |
| `fcf` | 自由现金流 | 原币种金额 | 建议 | 现金创造能力 |
| `fcf_margin` | 自由现金流率 | 比例 | 建议 | 现金质量 |
| `debt_to_equity` | 负债权益比 | 比例/倍数 | 是 | 杠杆与风险 |
| `share_count` | 期末股本 | 股数 | 是 | 稀释/回购 |
| `dividend_per_share` | 每股分红 | 原币种金额 | 建议 | 股东回报 |

### 4.2 可选增强字段

- `net_income`
- `operating_income`
- `roic`
- `buyback_amount`
- `capex`
- `eps`

说明：
- 如果可选字段抓取成本明显更高，可在第一轮先不补齐
- 第一轮优先确保最小字段集“抓对”

## 6. 字段级验真原则

### 6.1 必须双重校验

这些字段一旦出错，会直接污染后续研究：

- `current holder / is_current`
- `position_weight`
- `revenue`
- `roe`
- `roa`
- `debt_to_equity`
- `share_count`

校验方式：
- 原始来源核实
- 与辅助来源或手工计算交叉检查

### 6.2 抽样双查

- `gross_margin`
- `net_margin`
- `fcf`
- `fcf_margin`
- `dividend_per_share`

### 6.3 只作辅助参考

- 聚合站给出的描述性字段
- 无法回溯到官方口径的派生字段

## 7. 单位标准化规则

### 7.1 金额类

- 持仓原始值保留 `value_usd_thousand`
- 研究表统一标准化为 `value_usd`
- 展示层可再换算为 `亿美元`

### 7.2 比例类

- 内部统一用小数，例如 `0.153`
- 展示层再格式化为百分比

### 7.3 债务类

- `Debt/Equity` 必须记录原始口径
- 若来源使用百分比，转换后要显式注明

## 8. 异常检测规则

进入后续打分或分类前，至少检查：

- `position_weight < 0` 或 `position_weight > 1`
- `report_date` 与 `filing_date` 顺序异常
- 同一 `manager_entity + ticker + report_date` 存在重复冲突
- `roe`、`roa`、`gross_margin`、`net_margin` 超出合理区间
- `share_count` 与前后年度变化异常且无公司行为解释
- `debt_to_equity` 单位不一致
