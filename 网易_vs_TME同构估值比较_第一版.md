# 网易 vs TME 同构估值比较（第一版）

更新时间：2026-03-29

## 1. 结论先行

如果把 `网易` 和 `腾讯音乐 TME` 放进同一套估值语言里比较：

- 主方法：`主业 EV/EBIT`
- 主校验：`EV/FCF`
- 辅助校验：`DCF`
- 资产结构：`经营业务价值 + 净现金 = 股权价值`

那么第一版结论是：

- `从纯折价与安全边际看，TME 当前更便宜`
- `从确定性和大仓位舒适度看，网易更稳`
- `如果必须只选一个当前更合适的标的，第一版我偏向 TME`

这不是说网易不值得投，而是说：

- `在同构方法下，TME 当前给出的折价更深`
- 且 `保守情景` 下仍留出更明显的安全边际

## 2. 先把口径统一

为了避免“DCF 对 SOTP”这种跨方法硬比，当前统一采用以下比较口径：

1. 都用 `保守 / 基准 / 乐观` 三情景。
2. 都先算 `经营资产价值`，再单独加回 `净现金`。
3. 都用 `当前价 / 基准内在价值` 计算可比折价率。
4. 都用 `当前价 / 保守内在价值` 计算安全边际。

## 4. 折价率与安全边际

### 4.1 可比折价率

公式：

- `可比折价率 = 1 - 当前价 / 基准内在价值`

结果：

- 网易：`1 - 173.1 / 230.4 = 24.9%`
- TME：`1 - 9.43 / 17.06 = 44.7%`

### 4.2 安全边际

公式：

- `安全边际 = 1 - 当前价 / 保守内在价值`

结果：

- 网易：`1 - 173.1 / 194.7 = 11.1%`
- TME：`1 - 9.43 / 13.91 = 32.2%`

这说明：

- `TME 不仅基准折价更深，连保守情景下留出的边际也更厚`

## 5. 为什么 TME 会显得更便宜

第一，市场对 `TME` 的经营资产给的倍数很低。

- 当前价格隐含下，TME 的经营资产 `EV/EBIT` 只在一个很低的位置
- 这低于第一版同构估值里给出的保守 `10x`、基准 `12x`、乐观 `14x`

第二，TME 的资产负债表很轻，净现金对股权价值支撑更直接。

- `FY2025` 末现金类资产约 `RMB 38.04bn`
- 对一个轻资本平台来说，这部分安全垫的估值含义比普通内容公司更直接

第三，2025 年的扰动项反而让错杀更容易发生。

- `FY2025` 经营利润里包含一次性收益
- 市场容易对“剔除一次性项目后的真实盈利中枢”打折
- 这给了估值低位

## 6. 为什么网易虽然没那么便宜，但仍然更稳

第一，网易的估值底稿更厚。

- 现在网易已经有：
  - `SOTP`
  - `EV/EBIT`
  - `EV/FCF`
  - `DCF`
  - `非游戏业务`
  - `净现金`
- 所以它的估值结论可信度更高

第二，网易的现金流和股东回报可验证性更强。

- 游戏主业现金流厚
- 净现金更多
- 分红和回购对底部保护更明显

第三，TME 虽然更便宜，但它当前仍有几处模型精度不如网易：

- 2025 年报 / 20-F 还没完全补进同构底稿
- 在线音乐主业利润率还更多依赖归一化代理
- 轻资本平台的终值和倍数，仍然对假设更敏感

## 7. 第一版“绝对结果”怎么下

如果你要求现在就下第一版的绝对判断，我会这样表述：

- `赔率优先：TME`
- `确定性优先：网易`
- `综合投资性价比第一版结论：TME 略优于网易`

原因是：

1. `TME` 的基准折价率明显更深：`44.7%` vs `24.9%`
2. `TME` 的安全边际也更厚：`32.2%` vs `11.1%`
3. 这已经足以覆盖它相对网易稍弱的模型确定性

所以如果今天必须二选一，第一版我会排：

1. `TME`
2. `网易`

## 8. 但这个结论还有一个前提

当前这只是第一版同构比较，不是最终定案。

最值得继续补的只有两件事：

1. 把 `TME` 的 `2025 年报 / 20-F` 审计口径补齐，进一步收紧其经营现金流和分部质量判断。
2. 再做一版 `风险调整后折价`，把“折价深度”和“估值确定性”放进同一个分数里。

## 9. 当前可执行结论

如果你现在就要一个动作层面的答案：

- `想要更高赔率，优先看 TME`
- `想要更高确定性和更适合大仓位，优先看网易`
- `如果只允许给一个第一版答案，我给 TME`

## 11. 风险调整后的最终比较方法

为了把“谁更便宜”和“谁更稳”合成一个最终结论，我再加一层统一评分：

### 11.1 第一步：先算原始错杀分数

定义：

- `原始错杀分数 = 60% × 基准折价率 + 40% × 安全边际`

理由：

- 基准折价率反映主要赔率
- 安全边际反映下行保护
- 两者都重要，但赔率权重略高

计算结果：

- 网易：`60% × 24.9% + 40% × 11.1% = 19.4`
- TME：`60% × 44.7% + 40% × 32.2% = 39.7`

### 11.2 第二步：再乘估值确定性系数

定义：

- `风险调整后分数 = 原始错杀分数 × 估值确定性系数`

当前第一版确定性系数我按四个维度打分：

1. `业务可预测性`
2. `输入口径透明度`
3. `利润归一化难度`
4. `现金流与资产负债表可验证性`

对应结果：

| 项目 | 网易 | TME | 说明 |
| --- | ---: | ---: | --- |
| 业务可预测性 | 4.0 / 5 | 4.0 / 5 | 两者都不差，但网易更受产品周期影响，TME更受平台增长放缓影响 |
| 输入口径透明度 | 3.5 / 5 | 3.0 / 5 | 网易底稿更完整；TME 仍有部分口径依赖归一化代理 |
| 利润归一化难度 | 4.0 / 5 | 3.5 / 5 | TME 2025 有一次性收益，归一化难度更高 |
| 现金流可验证性 | 4.5 / 5 | 4.5 / 5 | 网易和 TME 都强，但网易当前底稿更厚 |
| 平均分 | `4.0 / 5` | `3.75 / 5` | 第一版研究判断 |
| 估值确定性系数 | `0.80` | `0.75` | 直接由平均分折算 |

### 11.3 风险调整后结果

| 项目 | 网易 | TME |
| --- | ---: | ---: |
| 原始错杀分数 | 19.4 | 39.7 |
| 估值确定性系数 | 0.80 | 0.75 |
| 风险调整后分数 | `15.5` | `29.8` |

## 12. 最终结论

按当前这版同构估值和风险调整后的结果：

1. `TME`
2. `网易`

也就是说，如果你现在要求我给出一个最终的一选一判断：

- `当前更合适的标的是 TME`

## 13. 为什么最终还是 TME 胜出

原因不是 TME 更稳，而是：

- `它便宜得明显更多`
- 这种折价深度已经足以覆盖它相对网易略弱的估值确定性

更直接地说：

- 网易是 `稳健但不算极端便宜`
- TME 是 `没那么稳，但便宜得足够多`

而在当前这版量化里，`便宜得足够多` 这一点占了上风。
