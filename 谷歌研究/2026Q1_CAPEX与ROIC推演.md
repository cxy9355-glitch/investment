# Alphabet CAPEX 能否转为巨额利润？—— ROIC 推演

## 一、先算清楚这钱花在了哪

| 项目 | FY2025 实际 | FY2026E（指引中值） | 增量 |
|---|---|---|---|
| CapEx | $91.4B | **$185B** | +$94B |
| 折旧（年度化） | ~$21B | ~$35-40B | +$15-19B |
| P&E 账面净值 | $247B | ~$380B⁴ | +$133B |

> ⁴ 推算：Q1 期末 P&E $281B + 剩余三季追加 ~$100B

**这笔钱主要流向**：
1. 数据中心土建 + 电力（~30-35%）— 折旧 20-30 年
2. 服务器/网络设备（~40-45%）— 折旧 4-6 年
3. TPU 自研芯片（~15-20%）— 折旧 4-5 年
4. 海底光缆等（余量）

---

## 二、CAPEX 到利润的传导路径（三条）

### 路径 1：Cloud 对外出租 — 最直接的变现

这是当前最清晰的变现路径。用 Q1'26 实际数据反推：

```
Cloud 季度营收: $20.0B
Cloud 季度利润: $6.6B (margin 32.9%)
Cloud 年化利润: ~$26.4B
Cloud 占用的累计 CapEx（估算）: ~$120-150B（历年基础设施累计）

Cloud ROIC ≈ 26.4 / 135 ≈ 19.5%
```

**19.5% 的 ROIC 相当不错**，但问题是这只是 Cloud 用了的那部分。随着 CapEx 翻倍，新投入的产能能不能维持/提升这个回报率？

**关键变量：AI 需求能否消化翻倍的产能**

目前 Cloud backlog $460B（翻了三倍），假设平均 4 年合同期，每年确认 $115B → Cloud 年化收入可以达到 $1150 亿。如果利润率维持 33%，就是 **$380 亿/年利润**。

```
FY2027-28 Cloud（backlog 消化情景）:
  营收: ~$115B
  营业利润: ~$38B（33% 利润率）
  占用 CapEx: ~$250B（累计）
  ROIC: 38/250 ≈ 15.2%
```

15% 的 ROIC 依然高于资本成本（Google WACC ~9-10%），**是创造价值的**。

### 路径 2：Search/YouTube 的 AI 化 — 间接变现

CapEx 也用于 AI 推理支持 Search 和 YouTube。这部分不直接产生收入，但：

```
Q1'26 Search 收入: $60.4B (+$9.7B YoY)
Q1'26 Services 利润: $40.6B (+$7.9B YoY)
```

假设 AI 贡献了 Search 增长的 50%（另一半是宏观和份额），AI 驱动增量收入 ~$4.8B/季 → 年化 ~$19B。这部分 CapEx 主要是推理算力而非训练，ROIC 很难单独核算但效果是明显的——Search 增速从 12%→19%，利润率不降反升（42.3%→45.3%）。

### 路径 3：自有芯片节省成本

Google TPU v5/v6 替代 NVIDIA GPU，每美元算力的成本优势约 30-40%（自研芯片不用付 NVIDIA 的 70%+ 毛利率）。

```
估算每年 TPU 替代节省:
  假设 Cloud 30% 的 AI 负载跑在 TPU 上
  对应 GPU 租赁成本 ~$15-20B/年
  节省 ~$5-7B/年 → 直接转为利润
```

这是纯成本端的利润改善，不依赖需求增长。

## 三、悲观情景：CAPEX 变沉没成本

如果以下情况发生，$185B 年度 CapEx 将血本无归：

**情景 A：AI 需求降温，算力过剩**
- Cloud backlog 只是"意向"，客户可能不兑现
- 数据中心建好后空置 → 折旧照计提但收入为零
- 历史上思科 2001 年的教训：互联网泡沫破灭后，电信运营商的光纤网络用了 10 年才消化

**情景 B：模型开源化，定价权丧失**
- Meta Llama、Mistral 等开源模型性能追平 Gemini
- 企业客户转向"用开源模型 + 自建服务器"，Cloud AI 的定价从每 token $0.01 跌到接近成本
- Google 的 TPU 优势被削弱（客户可以用 AMD/Intel 替代方案）

**情景 C：技术路径突变**
- 3 年内出现比 Transformer 架构更高效的模型（比如液态神经网络、脉冲神经网络）
- 现有 Transformer 优化的 TPU/GPU 集群在新技术面前效率大打折扣
- Google 投入 $400B+ 建的专门优化 Transformer 的基础设施面临提前减值

**情景 C 发生的概率最低但杀伤最大**——就像当年为 RISC 处理器优化的软件跑在 x86 上水土不服一样。

## 四、ROIC 定量推演

### 假设条件

| 参数 | 保守 | 基准 | 乐观 |
|---|---|---|---|
| Cloud 年度营收增速（未来 3 年平均） | 25% | 40% | 55% |
| Cloud 利润率 | 28% | 33% | 38% |
| AI 对 Search 的增量收入/年 | $10B | $20B | $30B |
| Search 增量收入的边际利润率 | 60% | 70% | 80% |
| TPU 替代节省/年 | $3B | $5B | $8B |
| 累计 CapEx（3 年） | $500B | $500B | $500B |

### 推演（3 年后，FY2028）

| | 保守 | 基准 | 乐观 |
|---|---|---|---|
| Cloud 营收 | $125B | $220B | $370B |
| Cloud 利润 | $35B | **$73B** | $141B |
| Search AI 增量利润 | $6B | $14B | $24B |
| TPU 节省 | $3B | $5B | $8B |
| **合计 AI 驱动增量利润** | **$44B** | **$92B** | **$173B** |
| — | — | — | — |
| 3 年累计新增折旧 | ~$120B | ~$120B | ~$120B |
| 非 AI 的存量利润³ | ~$80B | ~$80B | ~$80B |
| **FY2028E 总营业利润** | **~$5B** | **~$52B** | **~$133B** |
| **增量 ROIC⁴** | 8.8% | **18.4%** | 34.6% |

> ³ 假设非 Cloud/Search 业务利润维持 $80B（Services 非 AI 部分 + 其他）  
> ⁴ ROIC = 增量利润 / 累计 CapEx($500B)

### 解读

**保守情景（ROIC 8.8%）**：低于 WACC，CAPEX 在摧毁价值。发生条件：Cloud 增速降到 25%（竞争加剧/需求降温），且利润率回落。

**基准情景（ROIC 18.4%）**：创造价值但不算"巨额"。Cloud $73B 年利润 + Search AI $14B + 节省 $5B = $92B 增量。对应 18% 的 ROIC 高于资本成本，但不够惊艳。

**乐观情景（ROIC 34.6%）**：这才是"巨额利润"。需要 Cloud 维持 55%+ 增速 + 利润率提升到 38%，同时 AI 对 Search 的拉动达到 $30B/年。

## 五、历史上可类比的 CAPEX 超级周期

| 公司/时期 | 年度 CapEx | 通胀调整 | 结果 |
|---|---|---|---|
| 90 年代末电信运营商 | ~$100B/年 | ~$180B | **惨败**，产能过剩 10 年 |
| AWS 2010-2020 | ~$40B/年峰值 | ~$48B | **大胜**，ROIC >30% |
| 台积电 3nm 投资 | ~$36B/年 | ~$40B | **成功**，垄断定价权 |
| Google 2025-2028E | ~$185B/年 | ~$185B | **？** |

Google 的 $185B/年远超 AWS 当年的峰值。但 AWS 证明了一件事：**在需求确实存在的前提下，基础设施投资的回报可以非常高**。

区别在于：AWS 面对的是确定性极强的"企业上云"浪潮（每年 30-40% 行业增速），而 Google 面对的是 AI 需求——增速更快（Cloud +63%），但 **不确定性也更大**（AI 需求是否真实、竞争格局是否会变）。

## 六、核心判断

**CAPEX 能转为巨额利润的两个前提**：

1. **AI 不是泡沫** —— 企业客户对 AI 的需求是真实的、持续的（不是尝鲜式试用）。目前 Cloud backlog $460B 提供了很强的证据支持这一点。

2. **Google 的全栈优势能转化为定价权** —— TPU + Gemini + GCP 构成差异化（客户租的不只是算力，是"算力+模型+工具链"）。如果模型层被开源追平，TPU 的优势也就不在了。

**概率判断**：
- 基准情景（18% ROIC，创造价值但不算暴利）：**60%**
- 乐观情景（35% ROIC，真正的巨额利润）：**20%**
- 保守情景（ROIC < WACC，在摧毁价值）：**20%**

**最核心的监控指标**（决定 CAPEX 命运的三个数字，每季度盯住）：
1. **Cloud backlog** —— 如果从 $460B 开始下降，危险信号
2. **Cloud 利润率** —— 如果从 33% 回落，说明竞争在侵蚀定价权
3. **CapEx/OCF 比值** —— 如果持续 >70%，FCF 压力会越来越大，最终要么砍 CapEx，要么继续大规模发债

**一句话**：Google 在赌一个比"企业上云"更大的浪潮（企业 AI 化），赌注是 $500B+ 的累计 CapEx。$460B backlog 给了这场豪赌第一道安全垫，但距离开牌还有 2-3 年。
