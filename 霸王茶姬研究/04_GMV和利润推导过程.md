# GMV和利润推导过程

更新时间：2026-03-30

## 1. 本文档的用途

从现在开始，`霸王茶姬` 这条线只继续维护这一张主文档。

这张表只做一件事：

- 用同一条利润链，直接给出 `2025Q1 实际` 和 `2026Q1 测算` 的可对比结果

本文里的“利润”，统一指与 `2025Q1` 官方净利润尽量对齐的研究口径近似值。  
也就是说：

- `2025Q1` 用官方净利润
- `2026Q1` 不再单独拆财务费用和所得税，而是沿着历史实际利润链做近似推导

## 2. 当前结论

在目前这组假设下，我给 `2026Q1` 的中心利润值是：

- `5.1 亿元`

对应区间：

- `保守：4.3 亿元`
- `基准：5.1 亿元`
- `偏乐观：6.1 亿元`

和 `2025Q1` 官方净利润 `6.8 亿元` 相比：

- `同比约 -25%`

当前这版结论的核心含义是：

- `GMV` 还没有明显塌掉
- 但加盟端利润贡献在下行
- 直营扩张带来的利润增量，还不能完全抵消更高的总部费用

## 3. 统一利润链

后面统一按这一条式子推：

`利润 = 加盟端利润 + 直营端门店经营利润 - 总部费用`

其中：

- `加盟端利润`：不是官方直接披露项，是根据历史结果反推，或根据假设测算
- `直营端门店经营利润`：指直营门店层面的经营利润，不含总部费用
- `总部费用`：`其他运营成本 + 销售费用 + 管理费用`

## 4. 2025Q1 实际值先拆开

### 4.1 官方已经给出的数据

| 项目 | 2025Q1（百万元） | 说明 |
| --- | ---: | --- |
| 总GMV | 8,226.8 | 官方披露 |
| 直营收入 | 242.8 | 官方披露 |
| 直营门店经营成本 | 157.0 | 官方披露 |
| 其他运营成本 | 172.5 | 官方披露 |
| 销售费用 | 299.3 | 官方披露 |
| 管理费用 | 352.8 | 官方披露 |
| 净利润 | 677.3 | 官方披露 |

### 4.2 把 2025Q1 按利润链重排

先算直营端门店经营利润：

`242.8 - 157.0 = 85.8`

再算总部费用：

`172.5 + 299.3 + 352.8 = 824.6`

最后反推加盟端利润：

`677.3 + 824.6 - 85.8 = 1,416.1`

得到：

| 项目 | 2025Q1 实际（百万元） | 计算方式 |
| --- | ---: | --- |
| 加盟端利润 | 1,416.1 | `677.3 + 824.6 - 85.8` |
| 直营端门店经营利润 | 85.8 | `242.8 - 157.0` |
| 总部费用 | 824.6 | `172.5 + 299.3 + 352.8` |
| 利润 | 677.3 | 官方净利润 |

这一步最重要，因为它把 `2025Q1` 的真实利润链钉住了。  
后面 `2026Q1` 必须沿着同一条链来推，才能和 `2025Q1` 直接比较。

## 5. 2026Q1 GMV 先按平均营业门店数来算

### 5.1 为什么不用季度末门店数直接乘

Q1 的 GMV，本质上对应的是 `整季营业店日`，所以不能直接拿 `季度末门店数` 当整季都在营业。

这次我改成了更稳的处理：

- `2026Q1 季度平均营业门店数 = (2025Q3 末门店数 + 2026Q1 末门店数) / 2`

这不是最完美的办法，因为还缺 `2025Q4` 官方门店数。  
但它至少比“直接拿 2026Q1 期末门店数乘整季店效”更保守，也更符合第一性原理。

### 5.2 已知输入

这轮用到的核心输入是：

- 你给定的 `2026Q1` 单店月均销售额同比：
  - 国内 `-15%`
  - 海外 `-20%`
- 当前门店结构假设：
  - `2026Q1末国内门店 = 7,100`，其中直营 `500`
  - `2026Q1末海外门店 = 400`，其中直营 `250`

### 5.3 四个分部的 GMV 测算

| 分部 | 2025Q3末门店数 | 2026Q1末门店数 | 2026Q1平均营业门店数 | 2025Q1店均月GMV（元） | 2026Q1同比 | 2026Q1店均月GMV（元） | 2026Q1 GMV（百万元） |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 国内加盟 | 6,836 | 6,600 | 6,718.0 | 428,714 | -15% | 364,407 | 7,344.3 |
| 国内直营 | 240 | 500 | 370.0 | 570,190 | -15% | 484,661 | 538.0 |
| 海外加盟 | 135 | 150 | 142.5 | 325,791 | -20% | 260,633 | 111.4 |
| 海外直营 | 127 | 250 | 188.5 | 433,302 | -20% | 346,642 | 196.0 |

计算公式统一为：

`分部Q1 GMV = 平均营业门店数 × 单店月均GMV × 3`

合并之后：

| 项目 | 2026Q1 GMV（百万元） |
| --- | ---: |
| 加盟合计 | 7,455.7 |
| 直营合计 | 734.0 |
| 总GMV | 8,189.7 |

## 6. 2026Q1 利润详细推导

### 6.1 加盟端利润

这一步不再直接拿“报表收入口径”去套，而是沿着 `2025Q1` 的历史结果往下推。

`2025Q1` 反推出来的加盟端利润率约为：

`1,416.1 / 8,226.8 = 17.2%`

考虑到：

- 国内单店月均销售额同比 `-15%`
- 海外单店月均销售额同比 `-20%`
- 外卖平台补贴竞争仍在
- `2026-01-01` 之后分润机制变化还没有官方量化细节

这次基准情景把加盟端利润率下修到：

- `16.8%`

所以：

`加盟端利润 = 7,455.7 × 16.8% = 1,252.6`

即：

- `2026Q1 加盟端利润 = 1,252.6 百万元`

### 6.2 直营端门店经营利润

直营端这次按你要求，先把 `实收率` 压到更合理的位置，再往下拆。

基准假设：

- `直营实收率 = 80%`
- `直营毛利率 = 62%`
- `直营门店经营成本率 = 15.5%`

对应链条：

| 项目 | 2026Q1 基准（百万元） | 计算方式 |
| --- | ---: | --- |
| 直营GMV | 734.0 | 上一步测算 |
| 实收 | 587.2 | `734.0 × 80%` |
| 原材料成本 | 278.9 | `734.0 × (1 - 62%)` |
| 门店毛利收入 | 308.3 | `587.2 - 278.9` |
| 直营门店经营成本 | 113.8 | `734.0 × 15.5%` |
| 直营端门店经营利润 | 194.5 | `308.3 - 113.8` |

所以：

- `2026Q1 直营端门店经营利润 = 194.5 百万元`

换成对直营GMV的利润率，就是：

- `26.5%`

### 6.3 总部费用

总部费用这一步单独看，不压回加盟端，也不压回直营端。

基准假设：

- `总部费用率 = 11.5%`

所以：

`总部费用 = 8,189.7 × 11.5% = 941.8`

即：

- `2026Q1 总部费用 = 941.8 百万元`

### 6.4 最终利润合成

把三段合起来：

`利润 = 1,252.6 + 194.5 - 941.8 = 505.3`

即：

- `2026Q1 利润 = 505.3 百万元`
- `约 5.1 亿元`

## 7. 2025Q1 实际 vs 2026Q1 测算

| 项目 | 2025Q1 实际（百万元） | 2026Q1 测算（百万元） | 同比变化 |
| --- | ---: | ---: | ---: |
| 总GMV | 8,226.8 | 8,189.7 | -0.5% |
| 加盟端利润 | 1,416.1 | 1,252.6 | -11.5% |
| 直营端门店经营利润 | 85.8 | 194.5 | +126.7% |
| 总部费用 | 824.6 | 941.8 | +14.2% |
| 利润 | 677.3 | 505.3 | -25.4% |

这张表说明的其实是同一件事：

- `26Q1` 不是 GMV 崩了
- 真正拖累利润的是 `加盟端利润下滑 + 总部费用上升`
- 直营扩张带来的利润增量，方向是对的，但还不够大

## 9. 目前最影响结果的变量

如果只看这版模型，最重要的不是公式，而是下面三件事：

1. `加盟转直营` 的节奏  
因为这会同时影响加盟端利润和直营端利润，尤其会影响 `Q1 平均营业门店数`

2. `加盟端利润率`  
每变动 `1 个百分点`，利润大约变动：
- `7,455.7 × 1% = 74.6 百万元`

3. `总部费用率`
每变动 `1 个百分点`，利润大约变动：
- `8,189.7 × 1% = 81.9 百万元`

所以当前最值得继续补的数据，不是再去争一个漂亮的公式，而是：

- `2025Q4 官方门店数`
- `2026Q1 加盟转直营的实际落地速度`
- `2026-01-01` 后新分润模式的真实执行力度

这三件事一旦拿到，下一轮就继续直接在这张表里更新。
