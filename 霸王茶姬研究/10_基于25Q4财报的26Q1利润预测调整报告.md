## 一、先说结论

复核完以后，我认为上一版新增报告的问题确实比较严重，主要不是算错个位数，而是：

1. 在 `25Q4` 已经给出 `直营店 = 615 家` 的情况下，我仍然偷偷沿用了旧版拆分里的一组渠道效率比和区域拆分逻辑，导致 `26Q1` 的直营/加盟 GMV 拆分带入了过期假设。
2. 上一版里 `26Q1` 利润区间的基准值偏高，新的更稳中枢不应写成 `4.4 亿`，而应更接近：
   - `26Q1 调整后基准值 ≈ 4.2 亿元`

新的工作结论我改成：

- `偏保守情景：3.1 亿`
- `调整后基准情景：4.2 亿`
- `偏乐观情景：5.4 亿`

如果只保留一句话：

- `25Q4` 财报出来以后，旧版“`26Q1` 偏保守基准约 `5 亿`”已经站不住
- 新版更稳的中枢应放在 `4 亿出头`

## 二、这次纠错，具体错在哪里

### 2.1 错误一：在 `25Q4` 财报已经给出总直营店 `615` 家之后，仍然沿用旧版的隐含渠道效率比

上一版为了把区域 GMV 再拆成加盟/直营，我沿用了旧版主文档里隐含的一组比值：

- `国内直营单店月均 GMV / 国内加盟单店月均 GMV ≈ 1.33`
- `海外直营单店月均 GMV / 海外加盟单店月均 GMV ≈ 1.33`

这个做法的问题是：

1. 这组比值来自旧版模型，不是 `25Q4` 财报给出的新事实
2. `25Q4` 直营门店从 `367` 家跳到 `615` 家，经营结构已经明显变化
3. 在结构显著变化后继续沿用旧比值，会把旧假设伪装成新财报推导结果

这一步不该继续用。

### 2.2 错误二：上一版对 `26Q1` 的渠道 GMV 拆分路径不够干净

更合理的路径应该是：

1. 先用 `25Q4` 实际直营收入
2. 再结合你原模型里已经定义好的 `直营实收率 = 80%`
3. 先反推出 `25Q4` 直营 GMV
4. 剩余部分自然就是 `25Q4` 加盟 GMV
5. 再用 `25Q4` 实际加盟/直营店数去算两端各自单店月均 GMV

这样得到的 `25Q4` 渠道锚点，比上一版硬得多。

### 2.3 错误三：上一版给出的新基准 `4.4 亿` 偏高

上一版的基准情景算出的是：

- `437.1M`

这个数如果四舍五入到 0.1 亿，应该是：

- `4.4 亿`

但在本次纠错后，采用更干净的 Q4 渠道锚点，基准情景应为：

- `419.8M`

也就是：

- `4.2 亿`

## 三、这次重算采用的假设

### 3.1 保留的假设

这次我保留下面几项，不去硬改旧框架：

1. `直营实收率 = 80%`
2. `直营毛利率 = 62%`
3. 原模型给定的 `26Q1` 期末门店假设仍作为一个工作情景保留：
   - 加盟店 `6,750` 家
   - 直营店 `750` 家

这里要说清楚：

- 这不是公司指引
- 只是为了让新旧版本能在同一框架下对比

### 3.2 新增的纠错约束

这次我强制加入两个 `25Q4` 实际约束：

1. `25Q4` 实际总 GMV = `7,322.9M`
2. `25Q4` 实际直营收入 = `539.609M`

有了这两个约束，`25Q4` 的直营 GMV 和加盟 GMV 就不再需要靠旧版隐含比值去猜。

## 四、详细计算过程

下面所有数字都按这条顺序推：

`Q4 实际渠道 GMV -> Q4 实际单店月均 GMV -> Q1 平均门店数 -> Q1 渠道 GMV -> 加盟利润 + 直营利润 - 总部费用`

### 4.1 第一步：反推 `25Q4` 实际直营 GMV

已知：

- `25Q4` 直营收入 = `539.609M`
- `直营实收率 = 80%`

所以：

`25Q4 直营 GMV = 539.609 / 0.80 = 674.51125M`

### 4.2 第二步：反推 `25Q4` 实际加盟 GMV

已知：

- `25Q4` 总 GMV = `7,322.9M`

所以：

`25Q4 加盟 GMV = 7,322.9 - 674.51125 = 6,648.38875M`

### 4.3 第三步：算 `25Q4` 实际加盟/直营单店月均 GMV

已知：

- `25Q4` 加盟店数 = `6,838`
- `25Q4` 直营店数 = `615`

则：

`25Q4 加盟单店月均 GMV = 6,648.38875 × 1,000,000 / 6,838 / 3 = 324,090`

`25Q4 直营单店月均 GMV = 674.51125 × 1,000,000 / 615 / 3 = 365,589`

得到这张校准表：

| 项目 | 数值 |
| --- | ---: |
| `25Q4` 加盟 GMV | `6,648.4M` |
| `25Q4` 直营 GMV | `674.5M` |
| `25Q4` 加盟店数 | `6,838` |
| `25Q4` 直营店数 | `615` |
| `25Q4` 加盟单店月均 GMV | `324,090` |
| `25Q4` 直营单店月均 GMV | `365,589` |

这一步是本次纠错最关键的地方。  
从这里开始，后面的渠道锚点就来自最新财报，而不是旧版隐含比值。

### 4.4 第四步：算 `26Q1` 平均店数

保留你旧版的 `26Q1` 期末门店假设：

- `26Q1` 期末加盟店 = `6,750`
- `26Q1` 期末直营店 = `750`

结合 `25Q4` 实际：

- `25Q4` 期末加盟店 = `6,838`
- `25Q4` 期末直营店 = `615`

则 `26Q1` 平均店数为：

`加盟平均店数 = (6,838 + 6,750) / 2 = 6,794`

`直营平均店数 = (615 + 750) / 2 = 682.5`

### 4.5 第五步：设定 `26Q1` 单店月均 GMV 情景

基于 `25Q4` 的渠道锚点：

- 加盟单店月均 GMV：`324,090`
- 直营单店月均 GMV：`365,589`

给 `26Q1` 三个情景：

| 情景 | 加盟单店月均 GMV | 直营单店月均 GMV |
| --- | ---: | ---: |
| 偏保守 | `315,000` | `355,000` |
| 基准 | `330,000` | `370,000` |
| 偏乐观 | `340,000` | `385,000` |

这些假设分别对应：

1. 偏保守：Q1 没有明显季节性修复，甚至略弱于 Q4
2. 基准：Q1 出现小幅季节性修复
3. 偏乐观：Q1 修复更明显，但仍没有回到旧版高位

### 4.6 第六步：算 `26Q1` 渠道 GMV

公式：

`渠道 GMV = 平均店数 × 单店月均 GMV × 3 / 1,000,000`

#### 偏保守

`加盟 GMV = 6,794 × 315,000 × 3 / 1,000,000 = 6,420.33M`

`直营 GMV = 682.5 × 355,000 × 3 / 1,000,000 = 726.8625M`

`总 GMV = 6,420.33 + 726.8625 = 7,147.1925M`

#### 基准

`加盟 GMV = 6,794 × 330,000 × 3 / 1,000,000 = 6,726.06M`

`直营 GMV = 682.5 × 370,000 × 3 / 1,000,000 = 757.575M`

`总 GMV = 6,726.06 + 757.575 = 7,483.635M`

#### 偏乐观

`加盟 GMV = 6,794 × 340,000 × 3 / 1,000,000 = 6,929.88M`

`直营 GMV = 682.5 × 385,000 × 3 / 1,000,000 = 788.2875M`

`总 GMV = 6,929.88 + 788.2875 = 7,718.1675M`

汇总如下：

| 情景 | 加盟 GMV | 直营 GMV | 总 GMV |
| --- | ---: | ---: | ---: |
| 偏保守 | `6,420.3M` | `726.9M` | `7,147.2M` |
| 基准 | `6,726.1M` | `757.6M` | `7,483.6M` |
| 偏乐观 | `6,929.9M` | `788.3M` | `7,718.2M` |

### 4.7 第七步：设定利润链参数

这次使用以下三组参数：

| 情景 | 加盟端利润贡献率 | 直营门店经营成本率 | 总部费用率 |
| --- | ---: | ---: | ---: |
| 偏保守 | `16.0%` | `18.0%` | `12.5%` |
| 基准 | `17.0%` | `17.0%` | `12.2%` |
| 偏乐观 | `18.0%` | `16.0%` | `11.8%` |

其中直营利润仍沿用你原来的定义：

`直营门店经营利润 = 直营GMV × 80% - 直营GMV × 38% - 直营GMV × 门店经营成本率`

也就是：

`直营门店经营利润 = 直营GMV × (0.42 - 门店经营成本率)`

### 4.8 第八步：算三段利润

#### 偏保守

`加盟端利润贡献 = 6,420.33 × 16% = 1,027.2528M`

`直营端门店经营利润 = 726.8625 × (0.42 - 0.18) = 174.447M`

`总部费用 = 7,147.1925 × 12.5% = 893.3990625M`

`研究口径利润 = 1,027.2528 + 174.447 - 893.3990625 = 308.3007375M`

#### 基准

`加盟端利润贡献 = 6,726.06 × 17% = 1,143.4302M`

`直营端门店经营利润 = 757.575 × (0.42 - 0.17) = 189.39375M`

`总部费用 = 7,483.635 × 12.2% = 913.00347M`

`研究口径利润 = 1,143.4302 + 189.39375 - 913.00347 = 419.82048M`

#### 偏乐观

`加盟端利润贡献 = 6,929.88 × 18% = 1,247.3784M`

`直营端门店经营利润 = 788.2875 × (0.42 - 0.16) = 204.95475M`

`总部费用 = 7,718.1675 × 11.8% = 910.743765M`

`研究口径利润 = 1,247.3784 + 204.95475 - 910.743765 = 541.589385M`

## 六、我现在对这件事的判断

复核到这里，我认为更合理的表述应该是：

1. 旧版 `5 亿` 的单点结论不再适合作为偏保守基准。
2. 在 `25Q4` 最新财报约束下，`26Q1` 的研究口径利润中枢更接近：
   - `4.2 亿`
3. 如果 `Q1` 国内单店修复不明显，利润下看 `3 亿多` 是有可能的。
4. 如果 `Q1` 修复超预期、折扣池使用率更优、费用率回落更快，上看 `5.4 亿` 也仍然成立。

所以新的工作区间我建议你用：

- `3.1 亿 ~ 5.4 亿`

其中工作基准值：

- `4.2 亿`

## 七、后面最值得继续复核的变量

这次虽然把显眼错误纠掉了，但仍有 4 个变量必须继续盯：

1. `26Q1` 期末门店假设 `加盟 6,750 / 直营 750` 本身是否还需要再调
2. `直营实收率 = 80%` 是否仍适合 `25Q4` 后的直营结构
3. `直营毛利率 = 62%` 是否仍适合直营占比提升后的实际情况
4. `总部费用率` 在 `Q1` 到底更接近 `12.2%` 还是会继续抬高

如果后面继续迭代，我建议新的顺序是：

1. 先把 `26Q1` 期末门店假设再确认一遍
2. 再决定直营 `80% / 62%` 这两个参数是否要同步重估
3. 最后再更新利润区间
